# Contributing to xiacf

First off, thank you for considering contributing to `xiacf`! It's people like you that make the open-source community such a great place to learn, inspire, and create.

To ensure a smooth experience for both users and maintainers, please review the following guidelines before participating.

## 🐛 Bug Reports

We use GitHub Issues to track public bugs. If you find a bug, please open an issue and include the following information to help us resolve it quickly:

* A clear and descriptive title.
* A reproducible example (reprex) of the issue.
* The version of `xiacf` you are using.
* Your operating system and R version.

## 💡 Feature Requests

If you have an idea for a new feature or an enhancement, we'd love to hear about it! Please open an issue and clearly describe:

* The problem you are trying to solve.
* Your proposed solution or feature.
* Any relevant academic references or theoretical backgrounds, if applicable.

## 🛑 Pull Requests (Collaborators Only)

**Currently, we do not accept unsolicited Pull Requests from non-collaborators.**

As this package is closely tied to specific academic research and methodologies, maintaining the integrity and exact behavior of the C++ core engine is our top priority. 

* If you are not a designated collaborator, **please do not submit a Pull Request directly**. Unsolicited PRs will be automatically closed without review.
* If you find a bug or have a feature suggestion, please open an **Issue** first to discuss it with the maintainers. 

## 🤝 Code of Conduct

Please note that this project is released with a Contributor Code of Conduct. By participating in this project you agree to abide by its terms. Always be respectful and constructive in your interactions.

Thank you for your understanding and for helping make `xiacf` better through your feedback!
